# backup scripts

## なぜこのスクリプト群を書いたか

現職でGUI手順で行っていたパラメータバックアップをCLI化することで
再現性を高めた経験を元に作成。
人間のミスを前提にした設計を意識している。

## スクリプト一覧

| ファイル名 | 目的 |
|---|---|
| `backup_params.sh` | タイムスタンプ付きバックアップの作成・世代管理 |
| `backup_verify.sh` | 当日のバックアップ存在確認（冪等性の確保） |

## 設計で意識したこと

- `set -euo pipefail`：エラーの見逃し防止
- `trap cleanup EXIT`：成功・失敗問わず終了時に必ずログを記録する
- バックアップ元の存在確認：存在しないディレクトリを指定したら即座にエラー
- タイムスタンプ付きバックアップ：世代管理と追跡可能性の確保
- ログファイル出力：実行履歴をファイルに記録して後から追跡可能にする
- 世代管理（7世代保持）：ディスク枯渇を防ぐ
- 冪等性：何度実行しても同じ結果になる設計
- `mkdir -p`：冪等性の確保

## 使い方

### バックアップの作成
```bash
chmod +x backup_params.sh
./backup_params.sh <バックアップ元> <バックアップ先>
```

### バックアップの存在確認
```bash
chmod +x backup_verify.sh
./backup_verify.sh <バックアップ先>
```

## 実行例
```bash
# バックアップの作成
./backup_params.sh ~/practice/backup/source ~/practice/backup/dest

# バックアップの存在確認
./backup_verify.sh ~/practice/backup/dest
```

## 動作確認済み内容

- タイムスタンプ付きバックアップフォルダの作成
- ログファイルへの実行履歴の記録
- 7世代を超えた古いバックアップの自動削除
- バックアップ元が存在しない場合のエラー検知
- 当日バックアップが存在する場合のスキップ
- 当日バックアップが存在しない場合のエラー検知
